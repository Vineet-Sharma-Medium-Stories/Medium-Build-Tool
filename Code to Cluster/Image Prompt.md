**Prompt:**

Create a detailed, horizontal image generation prompt for a modern technical blog post header. The image should serve as a high-level architecture diagram or infographic for a "Code to Cluster" Kubernetes deployment pipeline on AWS.

**Overall Style and Theme:**
- **Theme:** "AWS Light Theme." A clean, professional, and modern design with a white or very light gray background. Use subtle AWS-inspired orange (#FF9900) as an accent color for arrows, highlights, and borders. The overall feel should be minimalist, technical, and easy to understand.
- **Format:** Landscape-oriented, high-resolution, suitable for a blog post hero image.

**Main Title:**
- Prominently display the title: **"Code to Cluster: Building a Bulletproof Kubernetes Deployment Pipeline on AWS"**
- The title should be at the top or center, using a modern, bold sans-serif font.

**Sub-title:**
- Below the main title, include the extracted subtitle from the story: **"A Practical Guide to CI/CD, Security, Secret Management, and Observability on Amazon EKS"**
- The sub-title should be in a slightly smaller, elegant font.

**Visual Flow (Left to Right):**
Create a clear, horizontal flow chart or pipeline from left to right that depicts the following stages. Use distinct, rounded rectangles or "nodes" for each stage. Use sleek, curved arrows (in the AWS orange accent color) with connectors to show the flow of the pipeline.

1.  **Developer Commit:** Start on the far left with an icon representing a developer and a "git commit" action. Include the **GitHub** logo (or a Git logo) to represent the code repository.
2.  **CI/CD Trigger:** Connect to a node representing the CI/CD orchestrator. Use the **AWS CodePipeline** logo and the **GitHub Actions** logo side-by-side within this node to show the available options.
3.  **CI Pipeline (Build & Test):** A larger node representing the Continuous Integration process. Inside this node, arrange the following technology logos in a small cluster:
    - **Trivy** logo (for security scanning)
    - **SonarQube** logo (for code quality)
    - **Docker** logo (for building the image)
4.  **Container Registry:** Connect to a node representing the artifact storage. Use the **Amazon ECR (Elastic Container Registry)** logo as the primary, with smaller **Docker Hub** and **Harbor** logos nearby to indicate alternatives.
5.  **Secret Management Layer:** A separate node or layer placed *above* the main pipeline, visually "injecting" secrets into the next stage. Use the **AWS Secrets Manager** logo and the **HashiCorp Vault** logo. Show a visual connector (a dotted line or an orange arrow) going from this node to the next.
6.  **CD Pipeline & Kubernetes:** A node representing the deployment to Kubernetes. Use the **Amazon EKS (Elastic Kubernetes Service)** logo prominently. Also include small icons or logos for **Helm** and **ArgoCD** to represent deployment tools.
7.  **Environments:** From the CD node, create two branches leading to two separate nodes: one labeled **"STAGING"** and another labeled **"PRODUCTION."** Use a clean branching arrow style.
8.  **Observability:** Below the Staging and Production nodes, place a larger node labeled **"Monitoring & Observability."** Include the **Prometheus** logo, the **Grafana** logo, and the **AWS CloudWatch** logo inside this node. Show a connector from the Environments to this node, and a connector from this node looping back to the "Deploy" stage with a label "Alerting / Rollback."

**Technology Logos:**
- Ensure all the mentioned logos are high-quality, official, and clearly legible.
- **AWS-specific logos:** AWS CodePipeline, Amazon ECR, Amazon EKS, AWS Secrets Manager, AWS CloudWatch.
- **Partner/Open Source logos:** GitHub, GitHub Actions, Trivy, SonarQube, Docker, Docker Hub, Harbor, HashiCorp Vault, Helm, ArgoCD, Prometheus, Grafana.

**Connectors and Aesthetics:**
- Use a consistent, clean style for all shapes and nodes.
- The connectors (arrows) should be smooth, curved, and have a consistent orange (#FF9900) color to match the AWS theme.
- Use subtle shadows to give the nodes a slight sense of depth.

**Author Credit:**
- Place the text **"Vineet Sharma"** in the bottom-right corner of the image. Use a professional, non-intrusive font.

**Overall Goal:**
The image should visually tell the story of a complete, secure, and robust CI/CD pipeline for Kubernetes on AWS, acting as a compelling and informative summary of the blog post's content.
