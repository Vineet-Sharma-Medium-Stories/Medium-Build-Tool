## 🗺️ How the Metromap Is Structured

The Python Metromap is organized like a real transit system. Understanding this structure is the key to navigating efficiently.

**The Main Lines**

There are four main lines in the Python Metromap, plus a terminal for capstone projects.

**Foundations Station (Series A)** – This is your entry point. Every passenger starts here. You learn the core building blocks: variables, data types, operators, control flow, and loops. Think of this as the central hub—all other lines connect through this station.

**Data Structures Express (Series C)** – This line runs parallel to Foundations but goes deeper. You master lists, tuples, dictionaries, and sets. You can take this line immediately after Foundations, or you can return to it later when you need a deeper understanding of specific data structures.

**Functions & Modules Yard (Series B)** – This is where you learn to organize code. Functions, arguments, return values, lambdas, recursion, modules, and packages. This line connects to every other line because every application needs functions.

**Object-Oriented Programming Line (Series D)** – This line teaches you to model the real world with classes and objects. Banking systems, vehicle fleets, payment processors—these require OOP. You can take this line after completing Functions & Modules Yard.

**File & Data Handling Line (Series E)** – This line covers reading and writing files, CSV and JSON processing, exception handling, context managers, and path management. Essential for any application that works with data.

**Advanced Python Engineering (Series F)** – This is the express train for experienced riders. Decorators, generators, iterators, memory management, testing, and type hints. Take this line after completing Foundations and Functions.

**Data Science & Visualization Line (Series G)** – This line branches off from Foundations and Data Structures. NumPy, Pandas, Matplotlib, Seaborn. Perfect for analysts, researchers, and anyone working with data.

**Web Development & Automation Line (Series H)** – This line connects Foundations, Functions, and File Handling to the web. Flask, Django, automation, web scraping, scheduling. For building APIs, web apps, and bots.

**AI & Machine Learning Line (Series I)** – This is the most advanced line. It requires Foundations, Data Structures, Data Science, and Advanced Engineering. Scikit-learn, TensorFlow, Keras, PyTorch. For building intelligent systems.

**Capstone Terminal (Series J)** – This is where all lines converge. You take everything you've learned and build complete, portfolio-ready applications.

## 🚉 Transfer Stations

Transfer stations are where concepts from one series connect to another. Knowing these connections helps you navigate efficiently.

**Foundations to Data Structures** – Variables and data types (Series A) are the foundation for understanding lists, tuples, dicts, and sets (Series C). You can transfer immediately after A.2.

**Foundations to Functions** – Variables and control flow (Series A) are prerequisites for writing functions (Series B). Transfer after A.5.

**Functions to OOP** – Functions (Series B) are the building blocks for methods in classes (Series D). Transfer after B.3.

**Data Structures to Data Science** – Lists and NumPy arrays (Series C) are essential for Pandas and data analysis (Series G). Transfer after C.1 and C.3.

**Functions to Web Development** – Functions and modules (Series B) are used extensively in Flask routes and Django views (Series H). Transfer after B.6.

**All Lines to Capstone** – Complete your chosen lines, then transfer to Series J to build integrated projects.

## 🚆 Local Routes vs Express Trains

**Local Routes** – Stop at every station. Every story in a series. This is for beginners or anyone who wants complete mastery. Takes longer but ensures no gaps.

**Express Trains** – Skip some stations. Only take the stories you need. This is for experienced programmers learning Python or anyone with specific goals.

**Example Local Route (Absolute Beginner)** – Series 0 (all 5 stories) → Series A (all 7) → Series B (all 6) → Series C (all 5) → Series D (all 6) → Series J (all 3). Total: 32 stories. Estimated time: 12-16 weeks.

**Example Express Route (Data Scientist)** – Series 0 (stories 1-3) → Series A (all 7) → Series C (stories 1, 3, 4) → Series G (all 5) → Series I (all 4). Total: 20 stories. Estimated time: 8-10 weeks.

**Example Express Route (Web Developer)** – Series 0 (stories 1-3) → Series A (all 7) → Series B (stories 1-3, 6) → Series H (all 5) → Series J (story 2). Total: 18 stories. Estimated time: 6-8 weeks.

## 🚦 When to Take Each Line

**Take Series A (Foundations) if** – You are new to Python or programming. This is mandatory for everyone.

**Take Series B (Functions) if** – You want to write reusable, organized code. This is mandatory for anyone building applications.

**Take Series C (Data Structures) if** – You work with collections of data. This is mandatory for data scientists and recommended for everyone.

**Take Series D (OOP) if** – You are building complex systems with multiple interacting components. Banking, games, simulations.

**Take Series E (File Handling) if** – Your application reads or writes files, logs, or configuration. This is most applications.

**Take Series F (Advanced) if** – You want to write production-quality code, optimize performance, or work on large teams.

**Take Series G (Data Science) if** – You analyze data, build reports, or create visualizations.

**Take Series H (Web Development) if** – You build web applications, APIs, or automation scripts.

**Take Series I (AI/ML) if** – You build intelligent systems, classifiers, or predictors.

**Take Series J (Capstone) if** – You are ready to build portfolio projects that demonstrate your skills.
