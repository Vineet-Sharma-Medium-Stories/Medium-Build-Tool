# ### Design Patterns in Microservices

| Pattern | Implementation | Benefit |
|---------|---------------|---------|
| **API Gateway** | YARP | Single entry point, cross-cutting concerns |
| **Service Discovery** | Consul, Kubernetes DNS | Dynamic service location |
| **Circuit Breaker** | Polly | Failure isolation |
| **Saga** | Distributed transactions | Consistency across services |
| **CQRS** | Separate read/write models | Optimized queries |
| **Event Sourcing** | Event store | Complete audit trail |
| **Sidecar** | Service mesh proxy | Observability, security |
